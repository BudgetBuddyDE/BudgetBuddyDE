# Developer documentation

This area contains the technical documentation for contributors and maintainers. It covers the repository, architecture, services, data model, quality checks, observability, and contribution workflow.

## Recommended reading path

1. Read [Development environment](development.md) to set up the repository.
2. Review [Architecture](architecture.md) and [Project structure](project-structure.md).
3. Use the component and service references while changing code.
4. Run the documented quality checks before opening a pull request.
5. Follow the contribution and release conventions for delivery.

## Technical areas

- **Development:** local setup, commands, database, and debugging
- **Architecture:** boundaries, authentication, data flows, and error handling
- **Components and packages:** reusable libraries and UI building blocks
- **Services and interfaces:** backend services, APIs, and contracts
- **Data:** schemas, migrations, and consistency rules
- **Quality and automation:** tests, linting, CI/CD, and releases
- **Observability:** logs, metrics, OpenTelemetry, and incident response

User-facing installation and operation instructions remain in the [User documentation](../user/index.md).
